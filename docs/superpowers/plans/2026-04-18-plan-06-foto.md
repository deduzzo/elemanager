# Plan 06 — Foto sezione

**Goal:** Upload di foto dai dispositivi degli editor (smartphone fotocamera nativa / desktop file picker) per ogni sezione, con metadata DB, preview gallery, lightbox, delete con permessi.

**Storage:** Supabase Storage (bucket privato + signed URL read). Compressione client-side.

## Modello dati

### Bucket `sezioni-photos`
- Privato (accesso via signed URL 1h)
- Path: `{giornata_id}/{sezione_id}/{yyyy-mm-dd}-{uuid}.{ext}`
- Max file size 5MB dopo compressione
- MIME allow: `image/jpeg, image/png, image/webp`

### Tabella `foto_sezione`

```sql
CREATE TABLE elemanager.foto_sezione (
  id uuid PK DEFAULT gen_random_uuid(),
  sezione_id uuid NOT NULL REFERENCES sezioni(id) ON DELETE CASCADE,
  elezione_id uuid REFERENCES elezioni(id) ON DELETE SET NULL, -- opzionale
  storage_path text NOT NULL UNIQUE,
  descrizione text,
  width int,
  height int,
  bytes int,
  uploaded_by uuid REFERENCES profiles(id),
  created_at timestamptz DEFAULT now()
);
```

## RLS

### foto_sezione
- SELECT: tutti authenticated
- INSERT: admin O editor se giornata.stato='open'
- UPDATE: admin O editor solo propri (descrizione)
- DELETE: admin O editor solo propri

### storage.objects (bucket sezioni-photos)
- SELECT: tutti authenticated (via signed URL)
- INSERT: admin O editor (controllato a livello app + RLS foto_sezione)
- DELETE: admin

## Compressione client

`browser-image-compression` (~30KB lib):
- maxSizeMB: 1 (target ~1MB per foto)
- maxWidthOrHeight: 2000px
- useWebWorker: true

## UI

### PhotoUploader component
- Input `<input type="file" accept="image/*" capture="environment" multiple>`
- Preview thumbnails before upload
- Progress bar per file

### PhotoGallery component

- Click thumbnail → Lightbox fullscreen

### Integrazione

## Task

### T1 — Migration 0008 + bucket + storage RLS

File: `supabase/migrations/0008_foto_sezione.sql`

```sql
-- Tabella foto
CREATE TABLE IF NOT EXISTS elemanager.foto_sezione (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  sezione_id uuid NOT NULL REFERENCES elemanager.sezioni(id) ON DELETE CASCADE,
  elezione_id uuid REFERENCES elemanager.elezioni(id) ON DELETE SET NULL,
  storage_path text NOT NULL UNIQUE,
  descrizione text,
  width int,
  height int,
  bytes int,
  uploaded_by uuid REFERENCES elemanager.profiles(id),
  created_at timestamptz NOT NULL DEFAULT now()
);
CREATE INDEX foto_sezione_sezione_idx ON elemanager.foto_sezione(sezione_id);
CREATE INDEX foto_sezione_elezione_idx ON elemanager.foto_sezione(elezione_id);

ALTER TABLE elemanager.foto_sezione ENABLE ROW LEVEL SECURITY;

CREATE POLICY foto_select ON elemanager.foto_sezione FOR SELECT TO authenticated USING (true);
CREATE POLICY foto_insert ON elemanager.foto_sezione FOR INSERT TO authenticated
  WITH CHECK (
    elemanager.is_admin() OR (
      elemanager.auth_role() = 'editor'
      AND uploaded_by = auth.uid()
      AND elemanager.giornata_is_open((SELECT giornata_id FROM elemanager.sezioni WHERE id = sezione_id))
    )
  );
CREATE POLICY foto_update ON elemanager.foto_sezione FOR UPDATE TO authenticated
  USING (elemanager.is_admin() OR uploaded_by = auth.uid());
CREATE POLICY foto_delete ON elemanager.foto_sezione FOR DELETE TO authenticated
  USING (elemanager.is_admin() OR uploaded_by = auth.uid());

-- Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE elemanager.foto_sezione;
ALTER TABLE elemanager.foto_sezione REPLICA IDENTITY FULL;

-- Storage bucket (idempotente)
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES ('sezioni-photos', 'sezioni-photos', false, 5242880, ARRAY['image/jpeg','image/png','image/webp'])
ON CONFLICT (id) DO UPDATE SET
  public = EXCLUDED.public,
  file_size_limit = EXCLUDED.file_size_limit,
  allowed_mime_types = EXCLUDED.allowed_mime_types;

-- Storage RLS
DROP POLICY IF EXISTS "sezioni_photos_read" ON storage.objects;
CREATE POLICY "sezioni_photos_read" ON storage.objects FOR SELECT TO authenticated
  USING (bucket_id = 'sezioni-photos');
DROP POLICY IF EXISTS "sezioni_photos_insert" ON storage.objects;
CREATE POLICY "sezioni_photos_insert" ON storage.objects FOR INSERT TO authenticated
  WITH CHECK (bucket_id = 'sezioni-photos');
DROP POLICY IF EXISTS "sezioni_photos_delete" ON storage.objects;
CREATE POLICY "sezioni_photos_delete" ON storage.objects FOR DELETE TO authenticated
  USING (bucket_id = 'sezioni-photos' AND (elemanager.is_admin() OR owner = auth.uid()));
```

Applica, aggiorna `database.types.ts`.

### T2 — Query hooks + storage utils

`src/lib/queries/foto.ts`:
- `useFotoBySezione(sezioneId, elezioneId?)` — realtime
- `useCreateFoto()` — mutation; signatures `{ sezione_id, elezione_id?, storage_path, width, height, bytes, descrizione? }` con uploaded_by = auth.uid()
- `useUpdateFoto()` — solo descrizione
- `useDeleteFoto()` — elimina DB row + file Storage
- `useSignedUrl(path)` — useQuery che chiama `supabase.storage.from('sezioni-photos').createSignedUrl(path, 3600)`

`src/lib/storage.ts`:
- `async uploadPhoto(file: File, giornataId: string, sezioneId: string) : Promise<{ path, width, height, bytes }>`
  - Genera nome: `${giornataId}/${sezioneId}/${new Date().toISOString().slice(0,10)}-${crypto.randomUUID()}.${ext}`
  - Carica via `supabase.storage.from('sezioni-photos').upload(path, file, { cacheControl: '3600', upsert: false })`
  - Ritorna metadata

## Sicurezza/Privacy

- Strip EXIF opzionale (future): la libreria compressione lo fa automaticamente in alcuni casi
- Signed URL 1h: l'URL non è permanente
- Path randomizzato (UUID): non enumerabile
- No GPS leakage: compressione rimuove EXIF di default
