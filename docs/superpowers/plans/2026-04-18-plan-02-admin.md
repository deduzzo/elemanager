# Elemanager — Plan 02: Admin CRUD Implementation Plan

> **For agentic workers:** use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Area admin completa: gestione utenti, giornate elettorali, elezioni, liste, candidati, import sezioni CSV, visualizzazione audit log.

**Architecture:** Feature-sliced sotto `src/features/admin/`. Query layer in `src/lib/queries/` con TanStack Query. UI primitives riusabili in `src/components/ui/`. Routing annidato `/admin/*` dentro l'AppShell esistente.

**Tech Stack:** React 18 + TanStack Query v5 + react-router-dom v6 + Supabase JS + Tailwind (dark/neon). Nuova dep: `papaparse` per parsing CSV client-side.

**Spec di riferimento:** `docs/superpowers/specs/2026-04-18-elemanager-mvp-fase1-design.md`, sezioni "User stories Admin", "Modello dati", "Import CSV sezioni", "Audit log".

**Prerequisiti:** Plan 01 completato (tag `plan-01-foundation-complete`). AuthProvider, useRole, AppShell, ProtectedRoute operativi. Schema DB + RLS applicati.

## Assunzioni UX

- **Creazione utenti**: avviene in Supabase Studio (admin aggiunge email/password). Nella nostra UI l'admin crea solo la **riga `profiles`** collegando all'UUID dell'utente auth. Il MVP non implementa invite email (scope Plan 05+).
- **Navigazione admin**: `/admin` mostra indice con 5 card (Utenti, Giornate, Sezioni, Audit). Ogni card link a sotto-pagina.
- **Giornata detail**: `/admin/giornate/:id` è un'unica pagina con tabs per ogni elezione. Ogni tab mostra form elezione + liste (con candidati espandibili inline).
- **Invito/revoca editor**: admin modifica `ruolo` del profilo (admin/editor/viewer) e `attivo` (true/false).

- Primary = `btn-neon` (già in tailwind); ghost = `bg-white/5 hover:bg-white/10 rounded-xl px-4 py-2`; danger = `bg-neon-pink/20 hover:bg-neon-pink/30 text-neon-pink rounded-xl px-4 py-2 border border-neon-pink/30`

### Input

- Props: `label: string`, `error?: string`, standard input attrs (tra cui `id`)
- Render: `<label>` con text, `<input>` con classi `mt-1 w-full rounded-xl bg-white/5 border border-white/10 px-3 py-2 focus:border-neon-cyan focus:outline-none focus:ring-2 focus:ring-neon-cyan/40`
- Se `error`, mostra sotto in `text-xs text-neon-pink mt-1`
- ID auto-generato se non fornito (usa `useId`).

### Select

- Simile a Input ma per `<select>`. Children = `<option>`.
- Stile coerente.

### Modal

- Props: `open: boolean`, `onClose: () => void`, `title: string`, `children: ReactNode`, `size?: 'sm'|'md'|'lg'` (default md).
- Overlay `fixed inset-0 bg-black/60 backdrop-blur-sm z-40 flex items-center justify-center p-4`
- Panel `glass-strong p-6 w-full max-w-md z-50 relative` (max-w varia con size).
- Header: titolo + bottone X close.
- Chiudere con ESC o click su overlay (non sul panel).
- Gestione focus trap basic: focus al primo input al mount, restore focus al trigger on unmount (opzionale ma utile).

### ConfirmDialog

- Wrapper di Modal per conferma distruttiva.
- Props: `open`, `onConfirm`, `onCancel`, `title`, `message`, `confirmLabel?` (default "Conferma"), `danger?: boolean`.
- Due bottoni: Cancel (ghost) e Conferma (danger se flag).

### PageHeader

- Props: `title: string`, `subtitle?: string`, `actions?: ReactNode`.
- Layout: flex row, titolo grande + subtitle a destra actions.

### index.ts

- Re-export di tutto: `export * from './Button'; export * from './Input'; ...`

**Pattern comune** per ogni entità (esempio `giornate.ts`):

```ts
import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query';
import { supabase } from '@/lib/supabase';
import type { GiornataRow, GiornataInsert, GiornataUpdate } from '@/lib/database.types';

const KEY = 'giornate';

export function useGiornate() {
  return useQuery({
    queryKey: [KEY],
    queryFn: async (): Promise<GiornataRow[]> => {
      const { data, error } = await supabase
        .from('giornate_elettorali')
        .select('*')
        .order('data', { ascending: false });
      if (error) throw error;
      return (data ?? []) as GiornataRow[];
    },
  });
}

export function useGiornata(id: string | undefined) {
  return useQuery({
    queryKey: [KEY, id],
    enabled: !!id,
    queryFn: async (): Promise<GiornataRow | null> => {
      if (!id) return null;
      const { data, error } = await supabase
        .from('giornate_elettorali')
        .select('*')
        .eq('id', id)
        .maybeSingle();
      if (error) throw error;
      return (data as GiornataRow | null) ?? null;
    },
  });
}

export function useCreateGiornata() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: async (input: GiornataInsert) => {
      const { data, error } = await supabase
        .from('giornate_elettorali')
        .insert(input)
        .select()
        .single();
      if (error) throw error;
      return data as GiornataRow;
    },
    onSuccess: () => qc.invalidateQueries({ queryKey: [KEY] }),
  });
}

export function useUpdateGiornata() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: async ({ id, patch }: { id: string; patch: GiornataUpdate }) => {
      const { data, error } = await supabase
        .from('giornate_elettorali')
        .update(patch)
        .eq('id', id)
        .select()
        .single();
      if (error) throw error;
      return data as GiornataRow;
    },
    onSuccess: (_d, v) => {
      qc.invalidateQueries({ queryKey: [KEY] });
      qc.invalidateQueries({ queryKey: [KEY, v.id] });
    },
  });
}

export function useDeleteGiornata() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: async (id: string) => {
      const { error } = await supabase
        .from('giornate_elettorali')
        .delete()
        .eq('id', id);
      if (error) throw error;
    },
    onSuccess: () => qc.invalidateQueries({ queryKey: [KEY] }),
  });
}
```

**Adatta per ogni entità** con:

- `profiles.ts`: `useProfiles()` (list tutti), `useCreateProfile()`, `useUpdateProfile()`, `useDeleteProfile()`. Tabella `profiles`. Nessun `useProfile(id)` perché il ruolo corrente è già servito da `useRole`.
- `giornate.ts`: come esempio sopra. Tabella `giornate_elettorali`.
- `elezioni.ts`: `useElezioniByGiornata(giornataId)` (filter su `giornata_id`), create/update/delete. Tabella `elezioni`. `queryKey: ['elezioni', giornataId]` per scope.
- `liste.ts`: `useListeByElezione(elezioneId)`, create/update/delete. Tabella `liste`. `queryKey: ['liste', elezioneId]`.
- `candidati.ts`: `useCandidatiByLista(listaId)`, create/update/delete. Tabella `candidati`. `queryKey: ['candidati', listaId]`.
- `sezioni.ts`: `useSezioniByGiornata(giornataId)`, `useBulkInsertSezioni()` (per CSV import, accetta array di `SezioneInsert` e fa upsert con onConflict `giornata_id, numero`), `useDeleteSezioniByGiornata(giornataId)` (per re-import clean).
- `audit.ts`: `useAuditLog({ limit?: number, tabella?: string })` con ordering `created_at DESC`, limit default 100. `queryKey: ['audit', { limit, tabella }]`.

### Query bulk

Nel file `src/lib/queries/sezioni.ts` aggiungi:

```ts
export function useBulkInsertSezioni() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: async ({ giornataId, rows }: { giornataId: string; rows: SezioneInsert[] }) => {
      const payload = rows.map((r) => ({ ...r, giornata_id: giornataId }));
      const { data, error } = await supabase
        .from('sezioni')
        .upsert(payload, { onConflict: 'giornata_id,numero' })
        .select();
      if (error) throw error;
      return data;
    },
    onSuccess: (_d, v) => qc.invalidateQueries({ queryKey: ['sezioni', v.giornataId] }),
  });
}
```
